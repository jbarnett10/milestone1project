Key Concepts

  • HTTP Semantics: Stable methods, status codes, headers, and URIs across versions.
  • Statelessness: Each request is independent; session state isn’t implied, which simplifies servers but requires cookies/sessions for continuity.
  • Persistent Connections: Reuse a TCP connection for multiple requests (HTTP/1.1), reducing handshake overhead and improving throughput.
  • Multiplexing: Parallel streams over one connection (HTTP/2; QUIC for HTTP/3), minimizing blocking when many assets load at once.
  • HPACK: Header compression algorithm in HTTP/2 that reduces repeated header bytes across requests.
  • QUIC: UDP‑based transport with integrated TLS 1.3 and stream multiplexing for HTTP/3.
  • Head‑of‑Line Blocking: Single lost TCP packet stalling others; mitigated by QUIC streams.
