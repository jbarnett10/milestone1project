Origins of HTTP: 0.9 and 1.0

The web began at CERN in the early 1990s, where Tim Berners-Lee proposed a distributed hypertext system. The first stack combined URLs, HTML, and HTTP, demonstrated on info.cern.ch. The WorldWideWeb browser/editor showed how HTTP could connect authors and readers through a simple request–response model.

HTTP/0.9 defined a single GET method that returned an untyped HTML payload. There were no headers and no content negotiation. HTTP/1.0 (1996) added request and response headers, media types, basic caching, and methods like HEAD and POST. Most servers still used short-lived connections (one request per TCP connection), which later motivated performance work in HTTP/1.1 and beyond.

These early versions defined the core request–response pattern and content typing that still shape the web. By introducing headers, methods, and media negotiation, HTTP/1.0 enabled richer pages and laid the groundwork for the later focus on efficiency and scalability.

Example: HTTP/1.0 Request

GET /index.html HTTP/1.0
Host: example.org

A simple HTTP/1.0 request with a Host header, which allowed multiple sites to share one server.
